## 使用脚手架创建express工程

安装 `express`的脚手架工具， `express-generator`:

```shell
npm install -g express-generator
```

查看 `express`的版本，确认已经安装好了 `express`

```shell
express --version
```

创建我的博客的后端工程文件

```shell
express mysite-backend
```

你可以选择更改默认监听的端口：

首先用 `vscode`打开 `mysite-backend`项目工程目录

### 修改默认端口号

打开 `./bin/www`配置文件，查找下面的代码,将默认端口修改为你想要监听的端口即可：

```js
var port = normalizePort(process.env.PORT || "3000");//可以在这里将"3000"更改为"3001"
```

### 添加启动提示

查看 `package.json`文件，可以看见项目启动命令为

```shell
npm start
```

 设置项目启动提示：查找下面的代码，添加下面注释的内容,这样每次启动项目，就可以直接根据中文判断项目是否成功启动。

```js
function onListening() {
  var addr = server.address();
  var bind = typeof addr === "string" ? "pipe " + addr : "port " + addr.port;
  debug("Listening on " + bind);
  //console.log("服务器已启动，正在监听3001端口");
}
```

浏览器地址栏输入 `http://localhost:3001`，即可看到 `Express Welcome to Express`的字样，表明服务器启动成功。

## 改造项目工程文件目录

安装好后项目后已有和需要创建的文件目录功能介绍：

* routes ：表现层
* service : （新建）业务逻辑层
* `dao` ： （新建）数据访问层
* public ： 静态资源目录

新建 service `dao` 文件目录，删除文件目录view

删除 `app` 与模板引擎相关的代码,即下面的代码：

```js
// view engine setup
app.set('views', path.join(__dirname, 'views'));
app.set('view engine', 'jade');
```



脚手架搭建工程入口文件（`app.js`）分析：

```js
// 引入第三方库
var createError = require("http-errors");
var express = require("express");
var path = require("path");
var cookieParser = require("cookie-parser");
var logger = require("morgan");
// 引入路由
var indexRouter = require("./routes/index");
var usersRouter = require("./routes/users");
// 创建服务器实例
var app = express();
// 使用中间件
app.use(logger("dev"));
app.use(express.json());
app.use(express.urlencoded({ extended: false }));
app.use(cookieParser());
app.use(express.static(path.join(__dirname, "public")));
// 使用路由中间件
app.use("/", indexRouter);
app.use("/users", usersRouter);

// catch 404 and forward to error handler
app.use(function (req, res, next) {
  next(createError(404));
});

// 错误处理中间件
app.use(function (err, req, res, next) {
  // set locals, only providing error in development
  res.locals.message = err.message;
  res.locals.error = req.app.get("env") === "development" ? err : {};

  // render the error page
  res.status(err.status || 500);
  res.render("error");
});

module.exports = app;
```

安装 `nodemon`在文件内容发生变化后自动重新启动服务器

`nodemon`的使用和配置教程见 [02.nodemon.md](..\..\express\02.nodemon.md) 

## 创建数据库连接实例

安装 `mysql`,启动服务，安装  `navicat`图形化界面工具。借助图形化界面工具创建 `mysite`数据库

安装 `sequelize`以及 `mysql`数据库的驱动，安装教程参考： [02.模型定义和同步.md](..\..\数据库\My SQL\数据驱动与ROM\02.模型定义和同步.md) 

新建文件 `/dao/dbConnect.js`，该文件负责创建数据库连接

先建立连接并进行连接测试（注意，进行测试时记得将数据库连接 `dbConnect.js`文件导入到入口文件 `app.js`中），内容如下：

```js
const { Sequelize } = require("sequelize");

const sequelize = new Sequelize("mysite", "root", "mlbzdx", {
  host: "localhost",
  dialect: "mysql",
  logging: false, // 输出日志到控制台
});

(async function () {
  try {
    await sequelize.authenticate();
    console.log("Connection has been established successfully.");
  } catch (error) {
    console.error("Unable to connect to the database:", error);
  }
})();
```

控制台输出 `"Connection has been established successfully."`表明连接成功。
